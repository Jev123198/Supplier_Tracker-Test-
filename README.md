## What is This Project About?

Imagine you work at a company that buys goods from several suppliers.
Some suppliers always deliver on time. Some are late. Some send the wrong quantity.

This project helps **track and score each supplier** based on how well they perform —
so the company knows who to trust and who needs improvement.

> Think of it like a **report card for suppliers** — just like grades in school! 🎓

---

## The Problem I'm Trying to Solve

A company has **6 suppliers** delivering products like rice, sugar, packaging boxes, and pallets.

The operations team wants to know:
- Are suppliers delivering **on time**?
- Are they sending the **full quantity** that was ordered?
- Are there **defects** (damaged or bad items) in the deliveries?
- Which suppliers are **reliable** and which ones are **at risk**?

Without tracking this data, the company is just guessing. This project turns that guess into **real data-backed decisions**.

---

## 💡 What I Built

I created **3 things** for this project:

### 1. 📄 Excel File — `Supplier_Performance_Tracker.xlsx`
A spreadsheet with 4 sheets inside:

| Sheet | What it contains |
|---|---|
| Delivery Data | 20 delivery records from Jan–Mar 2024 |
| Supplier Scorecard | Scores and grades (A/B/C/D) for each supplier |
| KPI Dashboard | Summary of key numbers and action items |
| How to Use | Guide to understand the file |

### 2. 🗄️ SQL File — `supplier_performance_tracker.sql`
A file with **10 SQL queries** that analyze the delivery data.

Each query answers a specific business question like:
- *"Which supplier has the worst on-time rate?"*
- *"How many days late was each delivery?"*
- *"Which category — Food, Packaging, or Logistics — performs best?"*

### 3. 📋 CSV File — `Suppliers_Clean.xlsx`
A clean version of the supplier master data
with duplicates removed and proper formatting.
